# AEM Asset Selector — Hosted Page

A self-contained, hosted web page that renders Adobe Experience Manager's **Asset Selector** micro-frontend. It lets content authors browse their AEM DAM and **drag-and-drop assets** directly into Adobe Target, Salesforce, Braze, or any web application — typically surfaced inside a Chrome extension's side panel.

> **Live URL:** [assets.vyingdigital.com](https://assets.vyingdigital.com)

---

## ✨ Features

| Feature | Description |
|---|---|
| **Guided Setup Wizard** | 4-step wizard walks users through IMS credential entry or support-ticket generation |
| **Local Credential Storage** | Credentials are stored in `localStorage` — never transmitted to external servers |
| **Adobe Asset Selector MFE** | Renders the official Adobe `PureJSSelectors` (UMD) with dark theme, multi-select, and detail panel |
| **Asset Tray** | Selected assets appear in a draggable tray at the bottom of the page |
| **Drag-and-Drop** | Assets can be dragged from the tray into any web app; exports `text/plain`, `text/uri-list`, `text/html`, and `application/json` data transfers |
| **Support Ticket Generator** | Auto-generates a ready-to-copy P2 Adobe support ticket for IMS provisioning |
| **Extension Messaging** | Listens for `AEM_SET_CONFIG` messages from a parent Chrome extension to auto-configure |

## 🏗️ Architecture

```
┌─ Chrome Extension (Side Panel) ─────────────────────┐
│                                                      │
│  iframe ──▶ assets.vyingdigital.com/index.html       │
│             │                                        │
│             ├── Setup Wizard   (credential entry)    │
│             ├── Asset Selector (Adobe MFE)           │
│             └── Drag Tray      (selected assets)     │
│                                                      │
│  postMessage ◀──▶ AEM_ASSET_DRAG_START / DRAG_END   │
└──────────────────────────────────────────────────────┘
```

The page is a **single static HTML file** (`index.html`) with no build step — it loads Adobe's Asset Selector SDK directly from `experience.adobe.com` at runtime.

## 🚀 Getting Started

### Prerequisites

- **AEM as a Cloud Service** access
- Adobe IMS credentials: `imsClientId`, `imsOrg`, `apiKey`
- Modern browser (Chrome recommended for extension support)

### Local Development

Since this is a purely static site, you can serve it with any HTTP server:

```bash
# Using Python
python3 -m http.server 8080

# Using Node
npx -y serve .

# Using PHP
php -S localhost:8080
```

Then open [http://localhost:8080](http://localhost:8080) in your browser.

### Configuration

On first launch the setup wizard will appear. You have two paths:

1. **I have credentials** → Enter your `imsOrg`, `imsClientId`, `apiKey`, and optionally `repositoryId` and `imsScope`.
2. **I need credentials** → Fill in your company/environment details and the wizard generates a ready-to-copy Adobe P2 support ticket requesting provisioning.

Credentials are persisted in `localStorage` under the key `aem-asset-selector-config`.

## 🔧 How It Works

1. **Boot** — On page load, the app checks `localStorage` for saved credentials. If found, it auto-launches the Asset Selector.
2. **IMS Auth** — Adobe's `PureJSSelectors.registerAssetsSelectorsAuthService()` handles the full OAuth/IMS flow in modal mode, redirecting back to the same origin.
3. **Asset Selection** — When assets are selected in the MFE, they are normalized and added to the tray.
4. **Drag-and-Drop** — Tray cards are `draggable`. On `dragstart`, the app sets multiple data-transfer types so target apps (Target, Salesforce, etc.) can consume the asset URL, metadata, or an `<img>` tag.
5. **Extension Integration** — A parent window (e.g., Chrome extension) can send `{ type: 'AEM_SET_CONFIG', config: {...} }` via `postMessage` to configure and auto-launch the selector without the wizard.

## 🌐 Deployment

This project is designed to be deployed as a **static site**. Any static hosting service works:

- **GitHub Pages** — push to the `main` branch and enable Pages in repo settings
- **Netlify / Vercel** — connect the repo; no build command needed
- **Adobe CDN / S3 + CloudFront** — upload `index.html`

> **Important:** The hosting domain (e.g. `assets.vyingdigital.com`) must be registered in Adobe's IMS redirect URL allowlist. This is done through the Adobe support ticket generated by the wizard.

---

## 🔒 Security Notes

- Credentials are stored **locally in the browser** (`localStorage`) and are never sent to any third-party server.
- IMS authentication is handled entirely by Adobe's SDK in modal mode.
- The page communicates with its parent frame via `postMessage` — always validate origins in production extensions.
